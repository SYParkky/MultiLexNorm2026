# MultiLexNorm2026 — SKKU AI Introduction Assignment

Multilingual lexical normalization system for the [MultiLexNorm 2026 shared task](https://noisy-text.github.io/2026/multi-lexnorm.html).  
Based on the winning approach of [Samuel & Straka (2021)](https://aclanthology.org/2021.wnut-1.54/) — fine-tuning ByT5 on synthetic + authentic normalization data.

## 🚀 Environment Setup

### 1. Clone & Create venv
```bash
git clone https://github.com/YOUR_USERNAME/multilexnorm2026
cd multilexnorm2026

# Windows
python -m venv multilexnorm_env
multilexnorm_env\Scripts\activate

# Mac/Linux
python -m venv multilexnorm_env
source multilexnorm_env/bin/activate
```

### 2. Install Dependencies
```bash
pip install -r requirements.txt
```

Contents of `requirements.txt`:
```
torch>=2.0.0
transformers>=4.40.0
datasets>=2.19.0
accelerate>=0.27.0
sentencepiece
huggingface_hub
tensorboard
```

> ⚠️ **PyTorch CUDA version note**: RTX 5080 requires CUDA 12.8.  
> If the above does not work, check [pytorch.org](https://pytorch.org/get-started/locally/) for the correct install command for your environment.

### 3. HuggingFace Login & Dataset Access
```bash
huggingface-cli login
# Enter your token (huggingface.co → Settings → Access Tokens)
```

Request dataset access once (approval required):
- https://huggingface.co/datasets/weerayut/multilexnorm2026-dev-pub

### 4. Train
```bash
# ByT5-base
python train/finetune_v4.py

# ByT5-large (RTX 5080 recommended)
python train/finetune_large_v2.py
```

> First run takes ~5 minutes for tokenization. Subsequent runs load from cache in ~5 seconds.

Cache directories:
- `data/cached` — ByT5 tokenized cache
- `data/cached_mt5` — mT5 tokenized cache

### 5. Evaluate
```bash
python evaluate/eval_err_fast.py --model_path ./byt5-base-finetuned/checkpoint-
```

### 6. Submit to Leaderboard
```bash
python evaluate/submit.py --model_path ./byt5-base-finetuned-v4-final
# Generates results/predictions/submission.zip
# Upload at: https://www.codabench.org/competitions/14162/
```

## 📈 TensorBoard — Monitoring Training Logs

You can monitor training in real time or review logs after training.

### Launch
```bash
# ByT5-base logs
tensorboard --logdir ./runs/logs_base_v4

# ByT5-large logs
tensorboard --logdir ./runs/logs_large

# Compare all runs at once
tensorboard --logdir ./runs
```

Open `http://localhost:6006` in your browser.

### Log File Location
```
runs/
├── logs_base_v4/    # generated by finetune_v4.py
└── logs_large/      # generated by finetune_large_v2.py
```

### Key Metrics

| Metric | Tab | Description |
|---|---|---|
| `eval/loss` | SCALARS | **Primary indicator for optimal epoch** — lowest point = best checkpoint |
| `train/loss` | SCALARS | Training progress — should decrease steadily |
| `train/grad_norm` | SCALARS | Training stability — spikes suggest lr is too high |
| `train/learning_rate` | SCALARS | Learning rate schedule |

> ⚠️ **Lower eval/loss does not always mean a better model.**  
> Always measure final performance using `eval_err_fast.py` for ERR score.  
> From our experiments: eval/loss and ERR do not always move together.

### Export Log Data
```bash
tensorboard --logdir ./runs --export_dir ./runs/exported
# Saves all scalar data as JSON under ./runs/exported/
```

## 📋 Augmentation Strategy

Based on per-language ERR analysis, we identified the following issues and strategies.

### Per-Language Analysis

| Language | ERR | Issue | Strategy |
|---|---|---|---|
| ko (Korean) | 8.43 | High FN — many non-standard words missed | Input-variation augmentation |
| ja (Japanese) | 10.54 | Mixed FN + FP | Input-variation augmentation |
| en (English) | 17.06 | High FN | Input-variation augmentation |
| de (German) | 9.74 | High FN | Input-variation augmentation |
| th (Thai) | 5.60 | **FP=399** — over-normalization | Add identity pairs (word → word) |
| id, vi, sr, sl | 40~70 | Performing well | No augmentation needed |

### Method 1: Input-Variation Augmentation (adopted)

Keep the target output fixed, only vary the input. Zero noise risk.

```python
# Original: "ㄹㅇ" → "정말로"
# Augmented variants:
"ㄹㅇㅋ"   → "정말로"   # trailing filler syllable
"ㄹㅇㄹㅇ" → "정말로"   # repetition
"ㄹ9"      → "정말로"   # number typo (ㅇ→0)
"ㄹㅇ..."  → "정말로"   # ellipsis
```

### Method 2: Identity Pairs (for Thai)

Add pairs where the input equals the output, so the model learns when **not** to normalize.

> ⚠️ **Watch the ratio**: too many identity pairs will hurt normalization performance.  
> Recommended ratio — identity:non-standard = 1:2 ~ 1:3

### Method 3: LLM-based Augmentation (optional)

Generating new (non-standard, standard) pairs using a local LLM is also an option.  
However, LLM-generated outputs may not match the ground truth labels, introducing noise.  
Currently considered as a supplementary method only.
